================================================================
網站設計交接檔  —  給 Claude Design（設計首頁與各頁面公版）
================================================================

專案名稱：Optimization 教學網站（暫名）
委託人：G
交接目的：
請依本檔設計「首頁」與各類「頁面公版(template)」的視覺與版面。
這是一個教數學最佳化(optimization)的教學網站，靈感來自
ccjou《線性代數啟示錄》(https://ccjou.wordpress.com/)，但要
現代化、支援互動動畫、可持續更新。

----------------------------------------------------------------
一、產品定位（設計時的核心約束）
----------------------------------------------------------------
- 讀者：作者自學複習 + 大學部 + 研究所學生。偏學術、重閱讀。
- 三大體驗價值：
    1) 淺顯易懂：概念先給直覺與圖，再進入形式化。
    2) 嚴謹證明：長篇數學公式與證明，證明區塊可摺疊展開。
    3) 動態視覺化：抽象概念配互動動畫（如 normal cone、凸集、
       梯度下降、單體法、分支定界樹）。
- 內容語言：中英混合。內文中文，專有名詞/公式/定理名英文。
  → 設計字體與排版必須同時照顧「中文(CJK) + 英文 + 數學公式」
    三者並存時的可讀性與行距。
- 風格關鍵字：乾淨、學術、易讀、不花俏、留白充足、長時間閱讀
  不疲勞。可參考 ccjou 的「內容為王」精神，但版面要更精緻現代。

----------------------------------------------------------------
二、技術約束（設計要能被實作，請務必遵守）
----------------------------------------------------------------
- 前端框架：Astro（內容為 Markdown/MDX）。
- 樣式：Tailwind CSS。請用 Tailwind 可表達的設計語彙
  （間距、字級、色階），避免難以落地的特殊效果。
- 數學排版：KaTeX（公式在建置時渲染為 HTML/CSS）。
- 互動視覺化：以 React island 嵌入，函式庫為 D3.js（幾何）
  與 Plotly（函數圖）。→ 文章頁需預留「互動元件容器」樣式
  （含載入中狀態、置中、可全螢幕、圖說 caption）。
- 留言：Giscus（GitHub Discussions），出現在文章頁底部。
- 搜尋：Pagefind（靜態搜尋），需要一個搜尋框與結果頁樣式。
- 必備：深色模式(dark mode)、響應式(手機/平板/桌機)、
  頁內目錄(TOC)、公式可複製、可摺疊證明區塊。

----------------------------------------------------------------
三、需要設計的介面清單（請逐一產出公版）
----------------------------------------------------------------
[全域元件 Global]
G1. 設計系統/公版：色票（亮色+深色兩套）、字體階層
    （CJK 內文字體 + 英文 + 等寬程式碼字體）、標題/內文/
    公式/程式碼的字級與行距、連結樣式、間距尺標、圓角/陰影。
G2. 頂部導覽列 Header：站名/Logo、主選單
    （首頁、學習路徑、各領域、每週問題、習題、速查、關於）、
    搜尋框入口、深色模式切換。需含手機版收合選單。
G3. 頁尾 Footer：版權、授權說明（可轉載須註明來源）、
    RSS、GitHub 連結、回到頂部。
G4. 側邊欄元件 Sidebar（文章頁與列表頁用）：本頁目錄(TOC)、
    近期文章、熱門文章、標籤雲、分類清單、月份歸檔。
    （這些是清單型小工具，請設計統一卡片樣式。）

[主要頁面 Pages / Templates]
P1. 首頁 Home：Hero 區（一句話介紹網站宗旨）、三條學習路徑
    入口卡、精選文章、最新文章、各領域快速入口。
P2. 文章頁公版 Article（最重要，長閱讀核心）：
    - 標題區：大標、麵包屑、metadata（領域徽章、類型徽章、
      標籤、閱讀時間）。
    - 內文排版：標題階層、段落、清單、圖片、表格。
    - 數學公式：行內公式與獨立置中公式(display)的樣式。
    - 可摺疊「證明」區塊：預設收合，展開看完整證明。
    - 互動元件容器：置中、圖說、載入中、可放大。
    - 程式碼區塊：語法高亮、可複製。
    - 提示框(callout)：直覺 / 注意 / 常見誤解 三種樣式。
    - 頁內目錄(TOC)：桌機固定在側，手機可收合。
    - 頁尾：延伸閱讀（相關文章卡片）、上一篇/下一篇、
      Giscus 留言區。
P3. 領域專題總覽頁 Domain Hub：某一領域（如 LP）的介紹 +
    該領域文章清單（可依類型/標籤篩選），呈現學習動線。
P4. 學習路徑頁 Learning Path：三條路徑（大學部入門 / 研究所
    進階 / 快速複習）各一個時間軸或步驟式版面，串連文章。
P5. 分類 / 標籤瀏覽頁 List：某分類或某標籤底下的文章列表
    （卡片或列表，含篩選與排序）。
P6. 每週問題頁 POW：
    - 列表版：歷來每週問題清單（依領域標籤篩選）。
    - 單題版：題目 + 可摺疊「解答」區塊 + 留言。
P7. 習題 / 考古題頁 Exercises：習題庫與研究所考古題入口與清單。
P8. 急救查詢 / 速查頁 Quick Reference：以「公式卡片 / 定義卡片」
    為主的速查版面，可快速掃視、依主題分組。
P9. 搜尋結果頁 Search：搜尋框 + 結果清單（標題、摘要、領域徽章、
    命中高亮）。
P10. 一般內容頁 Simple Page：關於、常見問題(FAQ)、留言板、
    作者隨筆專欄 共用的簡潔單欄版面。

----------------------------------------------------------------
四、設計時請特別注意
----------------------------------------------------------------
- 長篇數學閱讀的舒適度優先：內文寬度適中（約 65~75 字元）、
  行距寬鬆、公式與內文對比清楚。
- 中英數混排：避免中文與公式擠在一起，注意基線對齊。
- 徽章系統：領域(9 種)與類型(觀念/方法/證明/應用/導讀/答讀者問)
  各需一組可辨識的顏色徽章，深色模式也要好看。
- 互動元件要有「空狀態/載入中」樣式，因為動畫由 JS 後載入。
- 深色模式不是附加，請與亮色同步設計（尤其公式、程式碼、
  互動圖表的配色）。
- 全部要響應式；手機上 TOC、側邊欄、選單如何收合請一併設計。
