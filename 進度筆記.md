## 這是什麼
數學最佳化（optimization）教學網站。靈感來自《線性代數啟示錄》：一篇講透一個概念、交叉引用、數學嚴謹，但現代化 + 互動動畫 + 手機可讀。
三支柱：**淺顯易懂 → 嚴謹證明（可摺疊）→ 動態視覺化**。視覺風格：**書卷／學術期刊**（暖色紙感、單一磚紅主色）。

- **repo**：https://github.com/ccc888ccc/G_optimization （private 可）
- **正式網址**：https://g-optimization.pages.dev ✅ **已上線**（Cloudflare Pages，接 GitHub `main` 分支，push 後自動 build+部署，實測約 1 分鐘內生效）。

## 已定案決策
- **站名**：「G的呼吸小站」（副標 *Optimization, Illustrated*）→ 改站名只動 [`src/config.ts`](src/config.ts)。
- **徽章**：純文字 pill，**不加 icon**。
- **網域**：先用 pages.dev，暫不買。
- 文章網址：`/articles/[slug]`，slug 來自 frontmatter 的 `slug` 欄位（不是檔名）。
- 標籤雲字級規則未定 → 程式內留 `TODO(G)`。

- **三維度標籤對照**：[`src/lib/taxonomy.ts`](src/lib/taxonomy.ts)（DOMAIN_LABELS / TYPE_LABELS）。
- **徽章元件**：[`src/components/badges/`](src/components/badges/) 下 `DomainBadge.astro`（9 色填色 pill）、`TypeBadge.astro`（6 色外框 pill）、`TagChip.astro`。色值直接寫進 [`src/styles/global.css`](src/styles/global.css) 的 `.badge-domain-*` / `.badge-type-*` class（對照交接檔第 7 節）。
- **KaTeX**：`astro.config.mjs` 接了 `remark-math` + `rehype-katex`；另外寫了兩個自製 rehype plugin：
  - [`src/lib/rehype-eq-copy.mjs`](src/lib/rehype-eq-copy.mjs)：把置中公式（`.katex-display`）包成 `.eq` 容器，從 KaTeX 輸出的 MathML `<annotation>` 取出原始 LaTeX 存進 `data-tex`，供複製鈕使用。
  - [`src/lib/rehype-codeblock.mjs`](src/lib/rehype-codeblock.mjs)：把 Shiki 產生的 `<pre>` 包成 `.codeblock`（標題列顯示語言 + 複製鈕）。
  - ⚠️ **踩過的坑**：MDX/Markdown 裡寫 `$$公式$$`（單行）會被解析成**行內**數學，不會產生 `.katex-display`！要寫成三行：
    ```
    $$
    公式
    $$
    ```
    才會是置中公式（`.eq` 容器 + 複製鈕才會出現）。已修正 0.7 凸集那篇，之後寫文章記得用三行式。
  - Shiki 語法上色用 `theme: 'css-variables'`，但 Astro 7 會把變數改名成 `--astro-code-*`（不是官方文件常見的 `--shiki-*`），兩組都已在 `global.css` 定義，保險起見。
- **P2 文章頁**：[`src/layouts/ArticleLayout.astro`](src/layouts/ArticleLayout.astro) + [`src/pages/articles/[slug].astro`](src/pages/articles/[slug].astro)。含麵包屑、領域+類型徽章、標籤、byline、桌機固定 TOC + 手機抽屜（IntersectionObserver scroll spy）、延伸閱讀卡、上下篇、Giscus 佔位（真的要接的話改這個 div）。
  - `Callout.astro`（intu/warn/mis 三型）、`Proof.astro`（`<details>` 收合 + 三角旋轉 + ∎）都在 [`src/components/article/`](src/components/article/)。
  - **內容作者慣例**：如果文章要有「延伸閱讀」卡片，正文自己寫一個 `## NN 延伸閱讀` 標題起頭，layout 只會在 prose 之後自動接上 `.related` 卡片網格，不會自動生成標題（避免跟正文編號打架）。
- **互動 island（共 5 個，都在 [`src/components/islands/`](src/components/islands/)）**，全部包在 [`src/components/article/InteractiveFrame.astro`](src/components/article/InteractiveFrame.astro) 統一容器樣式裡：
  - `ConvexHullDemo.tsx`：拖曳點、d3-polygon 算凸包（0.7 凸集）。
  - `SeparatingHyperplaneDemo.tsx`：拖曳兩組點，用「最近點對的中垂線」構造分離超平面（0.12 分離超平面）——這個構造方式跟文章裡的證明是同一套，不是為了好看亂畫的。
  - `LPGeometryDemo.tsx`：滑桿轉動目標方向，等值線掃過凸五邊形，即時標出最佳頂點（1.3 LP 幾何）。
  - `SimplexPathDemo.tsx`：按鈕逐步 pivot，貪心走向相鄰更好的頂點，走到最佳解會顯示「已達最佳解」（1.5 單體法）——已用瀏覽器實際點擊驗證兩步後精確停在全域最佳頂點。
  - `StrongDualityDemo.tsx`：具體例子（max 2x+3y 對偶 min 4y₁+5y₂），兩邊各自切換頂點，數值上驗證弱對偶（dual ≥ primal 恆成立）與強對偶（兩者切到最佳時精確相等於 9）。
  - ⚠️ **踩過的坑**：React island 的初始 `useState` **不能用 `Math.random()`** 之類的非確定性值——SSR 算一次、client hydration 又算一次，兩邊對不上會出現 hydration mismatch（React 會整棵樹放棄 patch，事件也可能怪怪的）。每個 island 的初始座標都已改成寫死的常數，亂數只用在使用者主動觸發之後（純 client 端事件，沒有 SSR 這邊的問題）。**之後每加一個新 island，初始 state 都要記得這條**。
  - 全部用 `client:load`（不是 `client:visible`）；本地測試環境的 IntersectionObserver 觸發不穩，`client:load` 對這種小型 SVG demo 沒有效能負擔，比較保險。
  - `InteractiveFrame.astro` 有一個可選的 `slot="controls"` 給滑桿/按鈕列，但**目前 5 個 island 都沒用它**——因為滑桿要跟 SVG 共用 React state，所以控制列直接寫在 React 元件內部（回傳同一個 JSX 樹），不能拆到 Astro 的另一個 slot（那是完全獨立的渲染樹，狀態接不起來）。這個 slot 先留著，只適合不需要跟畫布共享狀態的靜態控制項。
- **MVP 內容全部完成**：`src/content/articles/` 下 7 篇，依《內容大綱與規劃.md》主幹順序，用 `prevSlug`/`nextSlug`/`related` 串起來，全部都是完整正文（定義/直覺 callout/互動 demo/證明/常見誤解/延伸閱讀）：`0.1-site-guide.mdx` → `0.2-problem-classification.mdx` → `0.7-convex-sets.mdx` → `0.12-separating-hyperplane.mdx` → `1.3-lp-geometry.mdx` → `1.5-simplex-method.mdx` → `1.11-strong-duality.mdx`。
  - 註：0.2 問題分類這篇因為不需要新 island、內容也偏「地圖式」介紹，仍保留較簡短的骨架式段落（有 TODO(G) 標記待日後擴寫細節），其餘 6 篇都是完整長文。

### ✅ 里程碑 9：教材銜接資料夾建好

為了讓「老師上課教材（PPT/PDF/課本）→ 我們的文章」這件事不會隨教材量增加而讓每次對話變重，建了 [`教材/`](教材/) 資料夾（規則細節見裡面的 `使用說明.txt`）：

```
教材/
  待處理/          # 使用者丟新教材進來的地方（不進 git，版權+體積考量）
  已處理/          # 已經轉化成文章的來源檔，歸檔用（不進 git）
  教材對照表.md    # 索引：主題 ↔ 教材出處 ↔ 對應文章 slug ↔ 狀態（進 git，唯一常看的檔案）
  使用說明.txt     # 操作流程 + 給 Claude 的固定規則
```

核心原則：每次只讀「這次要處理的那一段」（PDF 用 `Read` 工具的 `pages` 參數截取頁碼範圍，不整份塞進對話）；交叉引用舊文章用 `Grep`/`Read` 現查 `src/content/articles/`，不預先貼進對話；不逐字複製教材原文（版權規則，也正好符合「換成自己淺顯易懂的話寫」的目標）。

目前 `教材對照表.md` 是空的（尚無教材上傳），MVP 首批 7 篇是依原創大綱寫的，跟這份教材對照表無關，表裡已註記清楚避免混淆。
