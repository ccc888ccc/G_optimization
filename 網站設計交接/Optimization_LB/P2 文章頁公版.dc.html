梯度下降：在等高線上找到最低點

一句話直覺——站在山坡上，往最陡的下坡方向踏一步，重複到走不動為止，你就到了谷底。

G作者 G·2026 年 7 月 8 日·閱讀 18 分鐘·標籤 #梯度 #收斂 #line-search

在無約束最佳化裡，我們想找一個點 $x^\star$ 使目標函數 $f(x)$ 最小。梯度下降(gradient descent) 是最基本、也最能培養幾何直覺的方法：它只用到一階資訊——梯度 $\nabla f$。

01直覺：往最陡的下坡走

✦直覺梯度 $\nabla f(x)$ 指向函數上升最快的方向，所以它的反方向 $-\nabla f(x)$ 就是下降最快的方向。沿著它走一小步，函數值必定變小。

把 $f$ 想成一座山，等高線(contour)就是地圖上的等高圈。梯度永遠垂直於等高線、指向外圈；我們要做的，就是逆著它一路往盆地中心走。

02演算法與更新規則

給定起點 $x_0$ 與學習率(step size) $\eta>0$，反覆套用：

$$x_{k+1} = x_k - \eta\,\nabla f(x_k)$$

下面這個互動圖用一個細長的二次碗 $f(x,y)=0.15x^2+0.6y^2$ 演示它。拖動學習率滑桿，看收斂路徑如何改變：

▲注意學習率是成敗關鍵。太小收斂龜速；太大會在谷壁間反彈甚至發散。實務上常用線搜尋(line search)或自適應步長自動挑 $\eta$。

03收斂性：走得多快？

對凸(convex)且梯度 $L$-Lipschitz 連續的 $f$，取固定步長 $\eta=1/L$，梯度下降的函數值誤差以 $O(1/k)$ 收斂：

$$f(x_k) - f(x^\star) \le \frac{\lVert x_0 - x^\star\rVert^2}{2\eta k}$$
▸展開完整證明Proof · 無跳步

由 $L$-Lipschitz 梯度可得下降引理(descent lemma)：對任意 $x,y$，

$$f(y) \le f(x) + \nabla f(x)^\top (y-x) + \tfrac{L}{2}\lVert y-x\rVert^2.$$

令 $y=x_{k+1}=x_k-\eta\nabla f(x_k)$、$x=x_k$，並取 $\eta=1/L$，代入整理得

$$f(x_{k+1}) \le f(x_k) - \tfrac{1}{2L}\lVert\nabla f(x_k)\rVert^2.$$

再用凸性 $f(x^\star)\ge f(x_k)+\nabla f(x_k)^\top(x^\star-x_k)$，配合上式與 $\eta=1/L$，可證明

$$f(x_{k+1})-f(x^\star) \le \tfrac{1}{2\eta}\big(\lVert x_k-x^\star\rVert^2 - \lVert x_{k+1}-x^\star\rVert^2\big).$$

對 $k=0,\dots,K-1$ 求和，右式為望遠鏡級數(telescoping)，加上函數值單調遞減，即得

$$f(x_K)-f(x^\star) \le \frac{\lVert x_0-x^\star\rVert^2}{2\eta K}.$$

∎

若進一步假設 $f$ 為 $\mu$-強凸(strongly convex)，收斂會加速為線性(幾何)速率 $O(\rho^k)$，其中 $\rho$ 取決於條件數 $\kappa=L/\mu$。

04程式實作

用 NumPy 幾行就能寫出：

gradient_descent.py
import numpy as np

def gradient_descent(grad_f, x0, eta=0.1, iters=100, tol=1e-6):
    """固定步長梯度下降，回傳收斂軌跡。"""
    x = np.asarray(x0, dtype=float)
    path = [x.copy()]
    for k in range(iters):
        g = grad_f(x)
        if np.linalg.norm(g) < tol:      # 已到駐點
            break
        x = x - eta * g              # x_{k+1} = x_k - η ∇f
        path.append(x.copy())
    return np.array(path)
✕常見誤解「梯度為零就是最小值」——不一定。$\nabla f=0$ 只代表駐點(stationary point)，可能是鞍點或極大值。唯有在凸問題上，駐點才保證是全域最小。
