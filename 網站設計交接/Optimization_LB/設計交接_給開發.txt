要點：
- 只有一個主色(磚紅)。深色刻意偏暖、避免純黑，長讀不刺眼。
- 深色模式用 class 策略(html.dark 或根元素 .dark)，非 media。
  所有元件皆以 CSS 變數吃 token，切 class 即整站換色。

字體(Google Fonts)
- 中文內文 + 標題：Noto Serif TC (400/500/600/700) — 撐書卷感。
- 介面/導覽/徽章/圖說：Noto Sans TC (400/500/700)。
- 公式：KaTeX 內建 (Computer Modern)。
- 程式碼/數據/標籤：JetBrains Mono (400/500/600)。

字級/行距(長讀優先)
- H1 42 / H2 26 / H3 19 / 內文 17 / 引言 19 / 圖說 13.5 (單位 px)。
- 內文 line-height 1.9；內文欄寬約 65–75 字元(max-width ~720px)。
- 間距尺標：4 的倍數 (4/8/12/16/24/32/48/80)。
- 圓角：按鈕 4px / 容器·圖 8px / 徽章 100px。
- 卡片陰影：0 18px 40px -26px rgba(30,25,20,.4) + 1px hairline。

四、響應式斷點（已設計，見 頁面公版集 的「R 響應式收合」）
----------------------------------------------------------------
≥ 1024px 桌機　：內文＋固定 TOC 側欄雙欄；側邊欄小工具在右；
                 完整水平導覽。
640–1024 平板　：文章網格 3→2 或 2→1 欄；TOC 收合為頂部抽屜；
                 側邊欄移到內容下方。
< 640px 手機　 ：主選單 → ☰ 抽屜；TOC/側邊欄 → 可收合抽屜；
                 所有網格單欄堆疊；搜尋改全頁式。
共通：觸控目標 ≥ 44px；公式與表格 overflow-x 可橫向捲動；
      深/亮色套用同一組斷點。

----------------------------------------------------------------
五、技術落地備註（對應 G 已定案的技術方案）
----------------------------------------------------------------
- 框架 Astro，內容 Markdown/MDX；frontmatter 帶 domain / type /
  tags / paths，對應第二節三維度分類。
- 樣式 Tailwind：把第一節 token 放進 theme.extend；深色用 class 策略。
- 數學 KaTeX：remark-math + rehype-katex，建置時渲染。display 公式
  外層包一個容器 + 「複製 LaTeX」按鈕(把原始式存 data 屬性)。
- 互動視覺化：React island。文章頁需要「互動元件容器」統一樣式：
  標題列 + 置中畫布 + 圖說 caption + 載入中狀態 + (可)全螢幕。
  demo 幾何用 D3、函數圖用 Plotly。公版裡的梯度下降 canvas 只是
  容器樣式與行為的示範，正式版換成 island 元件即可。
- 可摺疊證明/解答/FAQ：語意上用 <details>/<summary>，預設收合
  (證明、FAQ 次題)；三角箭頭用 CSS 旋轉。
- 留言 Giscus(文章頁與單題頁底部)；搜尋 Pagefind(P9 結果頁樣式)。
- 程式碼區塊：語法高亮(kw/fn/str/num/com 五類色，見 G1)＋複製鈕。
- 圖片：公版用斜線佔位 + 等寬說明；正式版換真圖，記得深色也要好看。
