# Optimization 教學網站 — 內容大綱與規劃

> 靈感來源：ccjou《線性代數啟示錄》。目標是延續它「一篇講透一個概念、彼此交叉引用、數學嚴謹」的精神，同時補上它的三個缺點：設計陽春、缺乏圖片輔助、已停止更新。

---

## 1. 網站定位

- **核心讀者**：作者本人（學習＋複習）、大學部學生、研究所學生。
- **三大價值主張**
  1. **淺顯易懂**：每個概念先給直覺，再進入形式化定義。
  2. **嚴謹證明**：重要定理附上完整、無跳步的證明（可摺疊，讓初學者略過、進階者展開）。
  3. **動態視覺化**：抽象概念配互動／動畫圖，例如 normal cone、凸集分離、梯度下降、單體法路徑、分支定界樹。
- **語言**：中英混合。說明與敘述用中文；專有名詞、定理名稱、公式符號保留英文（首次出現時中英對照）。

## 2. 內容組織架構（解決「分類重疊」的核心設計）

不要用單一分類硬塞。採三個**互相獨立**的維度：

### 維度 A — 主領域（Domain）：每篇文章唯一歸屬，用於瀏覽
`基礎數學 / LP / IP‑BIP / NLP / SP / SIP / DP / 分解與對偶方法 / 工具與計算`

### 維度 B — 文章類型（Type）：用於篩選閱讀目的
`觀念(Concept) / 方法演算法(Method) / 定理證明(Proof) / 應用範例(Application) / 答讀者問(Q&A) / 導讀(Guide)`

### 維度 C — 標籤（Tag）：多對多，串起跨領域概念
例如 `對偶`、`KKT`、`凸性`、`Benders`、`normal cone`、`鬆弛`。一篇談 Benders 分解的文章主領域可放在「分解與對偶方法」，但同時標 `LP`、`SP`、`IP`，於是在這三個領域頁都會被交叉引用到。

> **原則**：一篇文章 = 一個主領域 + 多個標籤 + 可被多條學習路徑引用。重疊靠標籤和「延伸閱讀」交叉引用解決，不靠分類。

## 3. 完整內容大綱（依你給的優先順序：基礎 → LP → IP → NLP / SP / DP）

### 領域 0：數學基礎（Foundations）
先打穩地基，後面全部共用。
- 向量空間、矩陣、秩、值域與零空間、特徵值與特徵向量（快速回顧，連向 ccjou 風格）
- 正定／半正定矩陣、二次型
- 多變數微積分：梯度、Hessian、方向導數、Taylor 展開
- **凸集與凸函數**：凸組合、凸包、極點(extreme point)、多面體
- **分離超平面定理、支撐超平面定理**
- **凸錐、tangent cone、normal cone、polar cone** ← 動畫重點
- 機率統計基礎（為 SP／SIP 鋪路）：隨機變數、期望值、條件期望、情境(scenario)
- 最佳化問題的一般形式與分類、可行域、最佳解存在性(Weierstrass 定理)

### 領域 1：LP 線性規劃
- 標準式與幾何直觀（可行域是多面體）
- 基本可行解、頂點、基與非基變數
- **單體法(Simplex)**：pivot 運算、退化(degeneracy)、Bland's rule、循環問題 ← 動畫：頂點間移動
- 兩階段法 / 大 M 法（找初始可行解）
- **對偶理論(Duality)**：對偶問題構造、弱對偶、強對偶、互補鬆弛(complementary slackness) ← 動畫：對偶幾何
- 敏感度分析與參數規劃
- 內點法(Interior Point)簡介：障礙函數、central path
- 應用：運輸、指派、網路流

### 領域 2：IP / BIP 整數與 0‑1 規劃
- 整數規劃建模技巧：邏輯約束、固定成本、big‑M、indicator 變數
- LP 鬆弛(relaxation)與界限
- **分支定界(Branch & Bound)** ← 動畫：搜尋樹展開與剪枝
- **割平面(Cutting Plane)**：Gomory cut ← 動畫：逐步收緊可行域
- 分支切割(Branch & Cut)
- 全單模矩陣(totally unimodular)與自然整數性
- 拉格朗日鬆弛(Lagrangian relaxation)與拉格朗日對偶
- 應用：背包、集合覆蓋、TSP、排程

### 領域 3：NLP 非線性規劃
- **無約束最佳化**：一階／二階最優性條件、梯度下降、線搜尋(line search)、牛頓法、擬牛頓(BFGS)、共軛梯度 ← 動畫：等高線上的收斂路徑、牛頓 vs 梯度
- 收斂性與收斂速率分析
- **約束最佳化**：Lagrange 乘子、**KKT 條件**、約束規範(constraint qualification) ← 動畫：梯度落在 normal cone 的幾何
- 對偶理論、Slater 條件、對偶間隙(duality gap)
- **凸最佳化**：凸問題判定、標準類型(LP / QP / SOCP / SDP)
- 求解方法：罰函數法、障礙法、增廣拉格朗日、SQP
- 應用：最小平方、SVM、投資組合

### 領域 4：SP 隨機規劃
- 不確定性下的決策、期望值問題
- **兩階段隨機規劃(two‑stage with recourse)**
- 多階段隨機規劃與**情境樹(scenario tree)** ← 動畫
- 機會約束規劃(chance‑constrained)
- **L‑shaped 方法 / Benders 分解**
- 樣本平均近似(SAA)
- 風險度量：CVaR
- EVPI、VSS（完美資訊價值、隨機解價值）

### 領域 5：SIP 隨機整數規劃（Stochastic Integer Programming）
- IP × SP 結合的難點（不可用單純的對偶分解）
- 兩階段隨機整數規劃
- 整數 L‑shaped 方法、Benders 變體
- 對偶分解、progressive hedging
- 應用：隨機排程、產能規劃

### 領域 6：DP 動態規劃
- **最優性原理(Bellman's principle)** 與 Bellman 方程
- 有限期程 vs 無限期程
- 值迭代(value iteration)、策略迭代(policy iteration) ← 動畫：值函數收斂
- 隨機動態規劃、Markov 決策過程(MDP)
- 近似動態規劃(ADP) → 銜接強化學習
- 應用：庫存管理、最短路徑、資源配置

### 領域 7：分解與對偶方法（跨領域主題頁）
把散落各處的分解技巧集中導讀：Benders 分解、Dantzig‑Wolfe 分解、Lagrangian 對偶與 relaxation、progressive hedging。此頁大量交叉引用到 LP / IP / SP / SIP。

### 領域 8：工具與計算
- 建模語言與求解器：CVXPY、Pyomo、Gurobi、SciPy
- 計算複雜度、P / NP、NP‑hard 概念
- 數值穩定性與實作陷阱

## 4. 學習路徑設計（處理「優先順序」，與分類脫鉤）

同一批文章，依讀者不同串成不同路徑：

- **大學部入門路徑**：基礎（向量、微積分、凸集）→ LP（幾何、單體法、對偶）→ IP 入門 → NLP 無約束 → DP 入門。
- **研究所進階路徑**：凸分析（cone、分離定理）→ LP 對偶與內點法 → NLP（KKT、凸最佳化、對偶）→ SP → SIP → 分解方法。
- **快速複習路徑**：每領域只挑「觀念」與「定理速查」類文章，附公式卡片。
- **主題深潛路徑**（可自由擴充）：例如「對偶」一條線貫穿 LP 對偶 → Lagrangian 對偶 → SP 的 Benders。

## 5. 文章寫作模板（統一風格）

每篇文章建議固定骨架，讓初學者與進階者都能各取所需：

1. **一句話直覺** — 這個概念在解決什麼問題。
2. **動機／幾何圖像** — 配圖或動畫。
3. **形式化定義** — 中英對照。
4. **定理與證明** — 證明區塊預設可摺疊(展開/收合)，嚴謹無跳步。
5. **視覺化互動** — 抽象概念的動態演示。
6. **範例** — 手算小例子 + 程式碼(CVXPY/Python)。
7. **常見誤解 / 答讀者問**。
8. **延伸閱讀** — 交叉引用相關文章（靠標籤自動關聯）。

**數學排版**：用 KaTeX 或 MathJax（後續技術選型時決定）。
**中英混合規範**：專有名詞首次出現「中文(English)」，之後可只用其一並保持一致。

## 6. 需要動態視覺化的抽象概念清單（優先製作）

| 概念 | 所屬領域 | 視覺化重點 |
|---|---|---|
| 凸集 / 凸包 / 分離超平面 | 基礎 | 拖曳點看凸包、兩凸集被超平面分離 |
| normal cone / tangent cone / polar cone | 基礎、NLP | 在邊界點即時畫出對應的錐 |
| 梯度與等高線、梯度下降路徑 | NLP | 調整步長看收斂/發散 |
| 牛頓法 vs 梯度下降 | NLP | 同起點兩法路徑對比 |
| KKT 幾何 | NLP | 目標梯度 = 約束梯度的錐組合 |
| 單體法在多面體頂點移動 | LP | 逐步 pivot 動畫 |
| LP 對偶幾何 | LP | 原問題/對偶問題同步變化 |
| 分支定界搜尋樹 | IP | 樹展開與剪枝 |
| 割平面收緊 | IP | 可行域逐步逼近整數包 |
| 情境樹 | SP | 多階段展開 |
| 值迭代收斂 | DP | 值函數逐步逼近 |

**工具候選**：D3.js（互動幾何）、Plotly、p5.js、或預先算好的動畫。技術選型時一併決定。

## 9. 技術方案（已定案）

- **框架**：**Astro**。內容用 Markdown/MDX 撰寫（content collections），需要互動的地方以「島嶼(islands)」嵌入 React/JS 元件。兼顧好寫、效能與長期維護。
- **數學排版**：**KaTeX**（`remark-math` + `rehype-katex`）。渲染快、涵蓋絕大多數符號；建置時就轉好，不拖慢載入。
- **互動視覺化**：幾何類（凸集、cone、單體法路徑）用 **D3.js**；函數圖／等高線／收斂曲線用 **Plotly**；必要時輔以 p5.js。以 React island 封裝，在 MDX 內像 `<ConvexSetDemo />` 直接引用。
- **留言**：**Giscus**（基於 GitHub Discussions，靜態網站友善，符合「只要留言」的選擇）。訂閱與統計暫不做。
- **全站搜尋**：**Pagefind**（靜態、零後端，與 Astro 整合良好，支援中英）。
- **樣式**：Tailwind CSS + 深色模式；頁內目錄(TOC)、公式可複製、可摺疊證明區塊。
- **內容結構**：每篇文章 frontmatter 帶 `domain`（主領域）、`type`（文章類型）、`tags`（多標籤）、`paths`（所屬學習路徑），對應第 2 節的三維度分類。
- **部署**：**Cloudflare Pages（已定案，免費）**。Astro build 出的靜態檔，透過 GitHub repo 連動、push 自動部署；免費方案流量無上限、私人 repo 亦可、含免費 HTTPS 與自有網域。備選 Vercel（同為免費、CDN、私人 repo 可用）。純靜態站，不需要伺服器、Docker、後端或 .env。詳見同資料夾《部署方式.txt》。

> 完整的單篇文章清單見同資料夾的《完整文章大綱.md》。
